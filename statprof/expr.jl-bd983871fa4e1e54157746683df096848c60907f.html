121 """
122 @macroexpand
123
124 Return equivalent expression with all macros removed (expanded).
125
126 There are differences between `@macroexpand` and [`macroexpand`](@ref).
127
128 * While [`macroexpand`](@ref) takes a keyword argument `recursive`, `@macroexpand`
129 is always recursive. For a non recursive macro version, see [`@macroexpand1`](@ref).
130
131 * While [`macroexpand`](@ref) has an explicit `module` argument, `@macroexpand` always
132 expands with respect to the module in which it is called.
133
134 This is best seen in the following example:
135 ```julia-repl
136 julia> module M
137 macro m()
138 1
139 end

141 (@macroexpand(@m),
142 macroexpand(M, :(@m)),

145 end
146 end

151 end
152 @m (macro with 1 method)
153
154 julia> M.f()
155 (1, 1, 2)
156 ```
157 With `@macroexpand` the expression expands where `@macroexpand` appears in the code (module `M` in the example).
158 With `macroexpand` the expression expands in the module given as the first argument.
